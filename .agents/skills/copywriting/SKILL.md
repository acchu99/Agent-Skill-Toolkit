# Copywriting

> "Nobody reads ads. People read what interests them. Sometimes it's an ad." — Howard Gossage. Write to the reader's problem, not your feature list.

## When to Apply

Landing pages, product/pricing pages, feature descriptions, email sequences, cold outreach, in-app microcopy, launch announcements, ad creative, and content plans. Always finish with `writing-humanizer` so the copy doesn't read as AI-generated.

## Start With The Reader, Not The Product

Before writing a word, answer:

| Question | Why it matters |
|----------|----------------|
| Who is the reader? | One specific person, not "users". |
| What problem are they in right now? | Copy sells relief from a problem. |
| What do they believe today? | You move them from belief A to belief B. |
| What's the one action? | Every piece has exactly one primary CTA. |
| What's the objection? | Name it and answer it, don't dodge it. |

## Feature → Benefit → Outcome

Readers don't buy features. Translate every feature down the chain:

```text
Feature:  "256-bit AES encryption"
Benefit:  "Your data can't be read even if the disk is stolen"
Outcome:  "You pass the SOC 2 audit without a scramble"
```

Write the outcome in the headline, the benefit in the subhead, the feature in the details.

## Copy Frameworks

| Framework | Structure | Best for |
|-----------|-----------|----------|
| **PAS** | Problem → Agitate → Solution | Landing pages, ads, cold email |
| **AIDA** | Attention → Interest → Desire → Action | Long-form sales pages |
| **BAB** | Before → After → Bridge | Product pages, case studies |
| **4 Ps** | Promise → Picture → Proof → Push | Email, launch posts |
| **FAB** | Feature → Advantage → Benefit | Feature sections, spec tables |

## Headlines

- Lead with the **outcome or the specific number**, not the product name.
- Make it **falsifiable and concrete**: "Ship migrations without downtime" beats "The best database tool."
- Formulas that work: `[Achieve outcome] without [pain]` · `[Do X] in [timeframe]` · `The [tool] for [specific audience]` · `Stop [pain]. Start [gain].`
- Subhead does the second job: who it's for + how it works in one line.

## Landing-Page Copy Skeleton

1. **Hero:** outcome headline + one-line subhead + primary CTA + proof chip (logo/number).
2. **Problem:** name the reader's current pain in their words.
3. **Solution:** how it works, in 3 steps or 3 benefit blocks.
4. **Proof:** testimonials, metrics, logos, before/after.
5. **Objection handling:** pricing fears, switching cost, "will it work for me".
6. **Final CTA:** restate the outcome, remove risk (free trial, no card, guarantee).

## Voice & Tone

- Pick a **voice** (stable: e.g. plain-spoken, expert, playful) and vary **tone** by context (error message vs. celebration).
- Write at a **7th–8th grade reading level** for marketing; simpler converts better.
- Second person ("you"), active voice, present tense.
- One idea per sentence. Short sentences carry momentum.

## Email & Sequences

| Email type | Goal | Key move |
|------------|------|----------|
| **Welcome** | Set expectations, first value | Deliver one quick win |
| **Nurture/drip** | Build trust over time | One idea per email, one CTA |
| **Cold outreach** | Get a reply | Relevance + brevity; ask a question, don't pitch |
| **Re-engagement** | Win back dormant users | Remind of value, make return frictionless |
| **Launch** | Drive action in a window | Deadline + specific outcome |

Subject lines: specific > clever. Cold email: ≤ 90 words, one ask, no wall of features.

## Content Strategy (when the ask is broader than one page)

- **Content pillars:** 3–5 themes tied to what you sell and what the audience searches.
- **Topic clusters:** one pillar page + supporting articles, internally linked (see `seo-fundamentals`).
- **Editorial calendar:** cadence you can sustain > ambitious plan you'll drop.
- **Distribution plan:** decide the channel before writing (owned/email, search/SEO, social, communities).

## Anti-Patterns

❌ **Feature dumping** — a spec list with no outcome.
❌ **We/us focus** — "We built…", "Our platform…" instead of "You get…".
❌ **Vague superlatives** — "best-in-class", "world-leading" with no proof.
❌ **Burying the CTA** — reader finishes and doesn't know what to do.
❌ **Manufactured hype** — always run `writing-humanizer` after drafting.

## Quick Checklist

- [ ] Headline states an outcome, not the product name
- [ ] Every feature is translated to a benefit/outcome
- [ ] Exactly one primary CTA per page/email
- [ ] The main objection is named and answered
- [ ] Proof (numbers, logos, testimonials) is present
- [ ] Ran through `writing-humanizer`
